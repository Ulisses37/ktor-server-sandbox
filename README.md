## Ktor Server Sandbox (Phase 0)

This is a **learning** sandbox built to explore the Ktor framework and JetBrains Exposed (SQL ORM).

It serves as the backend prototype for a larger Git-Aware Sync Plugin I am building. The goal of this module was to master server-side Kotlin concepts before integrating them into the IntelliJ Plugin SDK.

### Project Goals

-**Build** a REST API in Kotlin using Ktor.

-**Implement Persistence** using SQLite and the Exposed framework.

-**Practice TDD** (Test Driven Development) with JUnit and HTTP Client tests.

-**Architecture**: Create a clean separation between the API layer (Routes) and Data layer (Repository).

### Tech Stack

**Language**: Kotlin

**Framework**: Ktor (Netty Engine)

**Database**: SQLite (Embedded)

**ORM**: JetBrains Exposed

**Build Tool**: Gradle (Kotlin DSL)

### How to Run

1. Clone the repository.

2. Open in IntelliJ IDEA.

3. Run the Application.kt file.

4. The server will start on http://0.0.0.0:8080.
